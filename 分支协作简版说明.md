# GitHub 分支协作简版说明

这个项目我们用同一个 GitHub 仓库，但每个人各自用自己的分支，互相独立维护，不强制合并。

## 1. 先下载仓库

```powershell
git clone https://github.com/liangrenjuan/ai-review.git
cd ai-review
```

## 2. 创建自己的分支

```powershell
git checkout -b 你的名字
```

例如：

```powershell
git checkout -b liangrenjuan
```

## 3. 开始修改

改代码、改文档、加数据都可以。

## 4. 提交并上传

```powershell
git add .
git commit -m "本次修改说明"
git push -u origin 你的名字
```

## 5. 每次开始前先同步

```powershell
git pull
```

## 6. 推荐习惯

- 开始改之前先 `git pull`
- 每个人只维护自己的分支
- 改完就提交，不要攒太久
- 先各自保留自己的版本，不需要合并到 `main`

## 7. 说明

如果以后想做最终整合，再单独处理合并。
现在这个协作方式就是“各改各的、各自同步、互不影响”。

如果不确定怎么合并，可以先发给我，我帮你处理。
