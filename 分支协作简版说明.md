# GitHub 分支协作简版说明

这个项目我们用同一个 GitHub 仓库，但每个人各自用自己的分支。

## 1. 先下载仓库

```powershell
git clone https://github.com/liangrenjuan/ai-review.git
cd ai-review
```

## 2. 创建自己的分支

```powershell
git checkout -b 你的名字
```

例如：

```powershell
git checkout -b liangrenjuan
```

## 3. 开始修改

改代码、改文档、加数据都可以。

## 4. 提交并上传

```powershell
git add .
git commit -m "本次修改说明"
git push -u origin 你的名字
```

## 5. 每次开始前先同步

```powershell
git pull
```

## 6. 推荐习惯

- 开始改之前先 `git pull`
- 每个人尽量用自己的分支
- 改完就提交，不要攒太久
- 尽量不要两个人同时改同一个文件

## 7. 最终怎么合并

如果某个版本要放到主分支，再把自己的分支合并到 `main`。

如果不确定怎么合并，可以先发给我，我帮你处理。
